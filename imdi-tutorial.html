First Steps (IMDI)

Learn how to use CMDI Maker for IMDI metadata generation

In the following tutorial, you will learn how to use CMDI Maker to generate IMDI metadata.

IMPORTANT: CMDI Maker saves all the data you input in a local browser database on your machine. You can close the website anytime without losing data. When recalling the app, you can pick up where you left off.
Your data is NOT uploaded to a server nor synced between brosers!

In order that CMDI Maker works correctly, it is necessary that your browser allows websites to store APPCACHE and DOM Storage data on your device. This is enabled by default.

1. Start

Start the app by going to http://cmdi-maker.uni-koeln.de.

You will see the start page where you can choose your profile, i. e. the workflow you want to use to generate metadata in the wanted XML format.
In the upper right corner, you can also change the language of the application.
After you have chosen your desired language and profile, click on "get started".

2. Corpus Information

In the workflow to generate IMDI files, the first thing to do is giving information about the corpus you want to create.
You can now begin with typing information into the input fields of the "Corpus" section.
When hovering with the cursor over an input field, a short description of the field will appear. This is intended for rendering assistance to the user. When you do not know straight what the field is for, this description may be of help.

After that, you can set global content languages for the whole corpus. When you type in a language name and click on search, a new view with search results will appear.

If the right language is present, you can add it to the corpus by clicking on it.
Alternatively, you can add languages by typing in the ISO639-3 code of the language, assuming that you know the right code. Delete languages by clicking on the "X" symbol of the specific language.
When finished, click on the next item in the workflow view, which is "Resources".

3. Resource Files

In the "Resources" view, you can choose resource files to be added to the corpus. Adding resources means, that CMDI Maker reads their filenames and filesizes and displays file objects that can be added to sessions later on.
There are two ways of importing resource files:
1. Drag files from your file system browser onto the hot area.
2. Click on "Choose files..." to open a file dialog and select your files there.

There are several types of files:

  • Written Resources like XML, EAF, TXT, ... files (green)
  • Media Files like WAV, MP4, MP3 (orange)
  • Invalid files (red)

    • Some file objects contain a warning. This is to inform you that these files are maybe not accepted by the LAMUS file uploader later on. Nonetheless, you can add all kinds of files (even invalid ones) to a session. It is possible to remove single files, to clear the whole file list and to sort files alphabetically. It is also possible to create one session per file. Before that you must choose which files of the list shall be added. There are several options:

  • Selected files (Files can be selected by clicking on them)
  • EAF files
  • WAV files
  • MPG files
  • MP4 files

    • When an EAF file is added to a session and the session has not been named yet, it gets named automatically after the EAF file name.
    When adding a file to a session, CMDI Maker also checks for files that start with the same string as the one to be added initially.

    Example:
    There are several resources avaliable:
    x001.eaf
    x001.wav
    x001_1.png
    y002.wav
    z003.eaf

    In the files menu, "EAF" is selected. In this case, two sessions will be generated:
    Session 1 will contain "x001.eaf", "x001.wav" and "x001_1.png". Session 2 will contain "z003.eaf".

    4. Actors

    Click on "Actors" to go to the next view.
    Add an actor by clicking on "New Actor". A new form will appear where you can fill in all information about one actor. If not sure what a field is about, a description will appear when hovering with the cursor over some fields. You can add actor languages the same way you added global content languages before. Here you can also choose if a language is the mother tongue and/or primary language of the actor.

    It is possible to delete actors, sort actors alphabetically and to duplicate actors.

    5. Sessions

    In the next view, sessions can be created and edited. It is possible to add single actors and resource files to a session. If you have created several sessions that share the same metadata, you do not have to manually type in the same metadata in every session. The function "Copy Session 1 metadata to all sessions" will take care of that. You can select the following metadata groups to be copied:

  • Date
  • Location
  • Project
  • Content
  • Actors

    • It is also possible to expand or collapse sessions, delete sessions and to sort them by name.
    Note that every session must have a name as well as a project name. Unnamed sessions are not allowed. Session names must contain only valid characters, these are [A-Z],[a-z],[0-9],"_".

    6. XML Output

    Congratulations! You mastered all steps to create IMDI metadata. When clicking on the last workflow item "XML output", you can marvel at your newly created XML. For the corpus and every session there is one XML file.

    It is possible to download single XML files as well as download them all at once or compress all to a ZIP archive.

    Settings

    There are several settings that can be adjusted. Go to the settings view by clicking on the hamburger menu icon in the upper right corner and then on "Settings".

    Settings

    Output Format

    Select in which output format the metadata shall be created: IMDI or CMDI with IMDI profile.

    Global Language Of Metadata

    The language of the metadata you have entered.

    CMDI Metadata Creator

    The CMDI metadata format requires the name of a metadata creator. This is probably you.

    Calculate Actor's Age

    When this feature is activated, CMDI Maker checks if the age of an actor (if it has not been specified already) can be calculated from the actor's birth date and the session date.
    When an age can be calculated, it will appear in the output file.
    (Age = Session Date - Actor's Birth Date)
